Primera aplicacion web creada con Django/Python para backend, solo HTML, CSS, Bootstrap y JavaScript para frontend y SQLite en desarrollo y PostgreSQL en producción

La app consiste en un formulario con los campos para escribir el nombre, apellido y el menu elegido del invitado. El diseño de la imagen de fondo no es de mi parte. Fue trabajo de un diseñador.

Utilizé Heroku como plataforma para desplegarla a producción. Solo estuvo activa 2 meses previos a la boda

A traves de un Event Listener puedo hacer que solo se pueda ingresar nombres apropiadamente escritos y no cualquier cosa. Si el usuario ingresa un numero o un caracter que no sea una letra del alfabeto el campo se delinea en rojo y el boton de submit ("confirmar asistencia") se deshabilita.

Si el usuario aprieta el boton de submit sin haber completado al menos un campo, aparecera un mensaje en rojo con una advertencia y no se agregara nada a la base de datos.

Algo similar sucede cuando algun usuario intenta agregar un nombre ya existente en la base de datos.

Cuando todos los campos son completados correctamente y el nombre no existe en la base de datos, aparecera un mensaje de exito y los datos se guardaran.

Usé el modelo predefinido de django "User" para guardar a los invitados ordenados alfabeticamente. En la imagen de abajo se puede observar la pagina de administrador de django con la lista de los invitados guardados. Tambien se puede notar que usé el campo de email para registrar el menú de cada uno

Para facilitar el acceso a esa misma lista por parte de la pareja u organizador, utilizé xlwt, una libreria de python para crear un archivo Excel con la lista de los invitados que confirmaron su asistencia y su respectivo menú. Tambien agregué la cantidad de invitados por menú.

El archivo se descarga a traves de un boton que esta protegido por contraseña para que no lo descargue cualquiera. El boton solo se habilita cuando la contraseña correcta es ingresada, sino permanece deshabilitado.
